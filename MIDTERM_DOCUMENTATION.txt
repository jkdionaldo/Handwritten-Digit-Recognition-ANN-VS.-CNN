4) ISSUES ENCOUNTERED AND HOW THEY WERE FIXED
==================================================
Issue 1: GUI closes immediately on startup
- Cause: cv2 was missing in the active terminal Python interpreter.
- Fix: Installed OpenCV in the same interpreter used by python command.
- Command used: python -m pip install opencv-python
- Verification: python -c "import cv2; print(cv2.__version__)"

Issue 2: GUI cannot load model file
- Cause: mnist_cnn.keras did not exist before training.
- Fix: Trained model first using train_mnist_models.py then launched GUI.

Issue 3: Lower prediction reliability on hand-drawn digits
- Cause: mismatch between user drawing style and MNIST input format.
- Fix: improved GUI preprocessing pipeline (crop, center, resize, normalize).

Issue 3b: Need easier model comparison for grading and demo
- Cause: single-model output made it hard to compare ANN vs CNN performance live.
- Fix: updated GUI to run both ANN and CNN on the same drawing and show both outputs side by side.

Issue 3c: UI looked awkward when confidence text became long
- Cause: long result strings caused layout stretching and uneven spacing.
- Fix: switched to minimalist fixed-size UI, compact result cards, and fixed-width confidence format.

Issue 4: Rubric mismatch for ANN flatten requirement
- Cause: ANN initially consumed already flattened data without a Flatten layer in architecture.
- Fix: ANN now accepts 28x28 input and includes layers.Flatten() in the model.

Issue 5: Search utility unavailable in terminal
- Cause: rg (ripgrep) command was not installed.
- Fix: Used PowerShell Select-String as fallback for code review and evidence checks.
